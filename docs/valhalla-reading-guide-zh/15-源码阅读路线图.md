# Valhalla 源码阅读路线图
## 第一阶段：建立整体心智模型
目标：知道在线链路和离线链路。推荐文件：`src/tyr/actor.cc`、`src/mjolnir/util.cc`。停止条件：能复述 Tyr->Loki->Thor->Odin。
## 第二阶段：读 route 主链路
目标：理解一次 route 如何跑完。推荐文件：`src/tyr/actor.cc`、`src/loki/route_action.cc`、`src/thor/route_action.cc`、`src/odin/worker.cc`。停止条件：能解释 Api 如何流动。
## 第三阶段：读图数据结构
目标：理解 GraphTile/GraphReader。推荐文件：`valhalla/baldr/graphtile.h`、`valhalla/baldr/graphreader.h`。停止条件：知道在线为何不直接读 OSM。
## 第四阶段：读路径搜索和 costing
目标：理解 Thor 如何使用 Sif。推荐文件：`src/thor/route_action.cc`、`valhalla/sif/dynamiccost.h`。停止条件：知道 costing 如何影响选路。
## 第五阶段：读瓦片构建
目标：理解 Mjolnir build stages。推荐文件：`src/mjolnir/valhalla_build_tiles.cc`、`src/mjolnir/util.cc`、`valhalla/mjolnir/graphtilebuilder.h`。停止条件：能说出 ParseWays 到 Build 的大流程。
## 第六阶段：读 map matching
目标：理解 trace 如何匹配到路网。推荐文件：`valhalla/meili/map_matcher.h`、`valhalla/meili/viterbi_search.h`、`test/mapmatch.cc`。停止条件：知道候选、转移、Viterbi 三个词。
## 第七阶段：读服务入口和 API 编排
目标：理解所有 action 如何复用 actor。推荐文件：`valhalla/tyr/actor.h`、`src/tyr/actor.cc`、`src/tyr/*_serializer.cc`。停止条件：能从 action 找到 Thor/Loki/Odin 分支。

## 每个阶段的目标、推荐文件和停止条件
见各阶段小节；每阶段只在达到停止条件后再进入下一阶段。

## 本章小结
源码阅读路线不是从目录 alphabetically 开始，而是从一次业务请求开始，再向图数据、算法和构建扩展。

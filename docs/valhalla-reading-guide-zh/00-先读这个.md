# Valhalla 源码阅读指南：先读这个
## 这份文档适合谁
适合有后端、导航、路线、ETA、地图服务经验，但第一次读 Valhalla C++ 源码的工程师。

## 这份文档不是什么
- 不是 API 文档
- 不是完整源码注释
- 不是算法论文
- 不是替代源码的材料

## 推荐阅读顺序
### 路线 A：我只想理解一次 route 请求
读 `01` -> `02` -> `03` -> Loki/Thor/Sif/Odin。
### 路线 B：我想理解 Valhalla 的图数据和路径搜索
读 `01` -> Baldr -> Sif -> Thor -> 调试指南。
### 路线 C：我想理解怎么从 OSM 构建 Valhalla tiles
读 `01` -> Mjolnir -> Baldr -> `14`。

## 如果你只有 2 小时
只读 `01-整体架构地图.md` 和 `03-一次route请求如何跑完.md`，并打开 `src/tyr/actor.cc:97-118` 对照。

## 如果你有 1 天
按 route 主链路读 Tyr、Loki、Thor、Sif、Odin，每章只读推荐文件的前两项。

## 如果你要深入改代码
先用测试复现，再读模块文档的“输入输出”和“和其他模块的关系”，最后改核心函数。

## 文档里的证据怎么看
证据格式是 `文件路径:行号` + 符号名 + 为什么能证明结论。行号可能随上游更新漂移，应以符号名二次确认。

## 遇到 TODO 怎么办
看到 `TODO: NEED HUMAN CONFIRMATION` 时，不要把它当结论；请按 TODO 表中的建议用源码、测试或运行结果确认。
